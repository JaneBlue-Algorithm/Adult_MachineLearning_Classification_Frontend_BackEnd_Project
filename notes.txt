# STEP 1 — Tek Bir Train/Test Split Kullan

Şu an notebook’ta her model için tekrar tekrar:

```python
train_test_split(...)
```

kullanıyorsun.

Örneğin:

* X_train_lr
* X_train_rf
* X_train_xgb
* X_train_svm

şeklinde birçok ayrı dataset oluşmuş.

Bu:

* kod tekrarına neden oluyor
* bakım zorlaşıyor
* model karşılaştırmasını zorlaştırıyor
* bazen veri tutarsızlığına yol açabiliyor

Yerine:

```python
X_train, X_test, y_train, y_test
```

şeklinde sadece 1 split kullan.

Bu en önemli düzenlemelerden biri.

---

# STEP 2 — Tek Bir Ortak Preprocessing Pipeline Kur

Şu an preprocessing bazı modellerde tekrar edilmiş.

Bunu tek bir yapı haline getir.

Önerilen yapı:

```python
categorical_features
numerical_features

preprocessor = ColumnTransformer(...)
```

ve bunu tüm modellerde kullan.

Ama burada önemli bir geliştirme daha var:

## Şu an eksik olan şey:

Eksik değer handling.

Şu anda:

```python
df.replace('?', 'Unknown', inplace=True)
```

kullanıyorsun.

Bu kötü değil.

Ama production seviyesinde daha doğru yöntem:

```python
SimpleImputer
```

kullanmak.

Örneğin:

```python
from sklearn.impute import SimpleImputer
```

Categorical:

```python
SimpleImputer(strategy='most_frequent')
```

Numerical:

```python
SimpleImputer(strategy='median')
```

---

# STEP 3 — Pipeline Yapısını Tam Profesyonel Hale Getir

Şu an bazı modeller pipeline içinde.
Bazıları değil.

Tüm modeller şu yapıda olmalı:

```python
Pipeline([
    ('preprocessor', preprocessor),
    ('model', model)
])
```

Bunun avantajı:

* data leakage önlenir
* deployment kolaylaşır
* backend entegrasyonu kolaylaşır
* model save/load işlemleri kolaylaşır

# STEP 5 — Cross Validation Kullan

Şu an modeller tek split ile test ediliyor.

Bu bazen şanslı/şanssız sonuçlar verebilir.

Daha profesyonel yöntem:

```python
cross_val_score
```

veya:

```python
StratifiedKFold
```

kullanmak.

Örnek:

```python
from sklearn.model_selection import cross_val_score
```

Bu gerçek performansı daha doğru gösterir.

# STEP 6 — Hyperparameter Tuning (EN BÜYÜK GELİŞİM)

Şu an modeller default parametrelerle çalışıyor.

Gerçek performans artışı tuning ile gelir.

Özellikle:

## Random Forest

```python

max_depth
min_samples_split
min_samples_leaf
```

## XGBoost

Tuning yapılmalı:

```python

colsample_bytree
```

Kullan:

```python
RandomizedSearchCV
```

# STEP 8 — Feature Engineering

Şu an en büyük eksiklerden biri bu.

Adult Income dataset feature engineering için çok uygun.

Örnekler:

```python
df['has_capital_gain'] = (df['capital_gain'] > 0).astype(int)
```

# STEP 9 — Outlier Analysis

Şu an outlier handling eksik.

Özellikle:

* capital_gain
* capital_loss
* fnlwgt

çok skewed olabilir.

Şunları deneyebilirsin:

```python
np.log1p()
```

Örneğin:

```python
df['capital_gain'] = np.log1p(df['capital_gain'])
```

Bu özellikle Logistic Regression ve SVM’de fayda sağlar.
